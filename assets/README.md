# PD Portfolio VS Code Kit

이 폴더를 Visual Studio Code로 열면 바로 수정할 수 있는 정적 포트폴리오 웹페이지입니다.

- `index.html`: 화면에 보이는 문구와 섹션 구조
- `styles.css`: 색감, 폰트, 간격, 반응형 디자인
- `assets/hero-prism-studio.png`: 첫 화면 배경 이미지
- `planning-baerini-video.html`: 유튜브 업로드 영상 `배린이가 미치면 생기는 일` 기획안

## 빠른 수정 순서

1. `지원자 이름`, 이메일, 전화번호, 링크를 실제 정보로 교체합니다.
2. `Core Pitch`, `Experience Focus`, `CASE DECK`을 본인 경험에 맞게 먼저 채웁니다.
3. 지원 회사가 바뀔 때마다 `TARGET FIT`의 회사명, 사업영역, 내가 맞는 이유를 바꿉니다.
4. 브라우저에서 `index.html`을 열어 문구 길이와 모바일 화면을 확인합니다.

## 섹션별 작성 가이드

### 1. Hero

첫 화면은 "나는 어떤 PD인가"를 한 문장으로 각인시키는 영역입니다. 감성적인 문구만 쓰기보다, 지원 직무와 바로 연결되는 행동을 넣는 것이 좋습니다.

좋은 구성:

- 1줄: 나의 핵심 역할
- 2줄: 콘텐츠를 어떤 결과로 연결하는지
- 본문: 기획, 촬영, 편집, 성과 관점 중 가장 강한 역량 2~3개

예시:

> 크리에이터의 매력을 발견하고, 브랜드가 이해할 수 있는 콘텐츠 구조로 설계합니다.
> 숏폼 기획부터 촬영 운영, 편집 방향, 썸네일 카피까지 콘텐츠가 조회수와 전환으로 이어지는 흐름을 만듭니다.

### 2. Recruiter Brief

채용 담당자가 10초 안에 읽는 요약 영역입니다. 추상적인 성격 장점보다 "바로 맡길 수 있는 일"을 적는 것이 전문적으로 보입니다.

추천 항목:

- 업무 포지션: 콘텐츠 PD, 크리에이터 비즈니스 PD, 숏폼 PD 등
- 핵심 강점: 리서치, 기획안, 질문지, 촬영 진행, 편집 디렉션
- 회사 적합성: MCN, 커머스, 브랜드 협업, 커뮤니티 운영 중 연결 가능한 영역
- 확인 포인트: 아래 포트폴리오에서 볼 수 있는 실제 결과물

예시:

> 크리에이터의 캐릭터와 브랜드 메시지를 연결하는 콘텐츠 PD입니다. 트렌드 리서치, 대본 구성, 질문지 제작, 현장 진행, 편집 피드백까지 제작 흐름 전체를 이해하고 움직입니다.

### 3. Resume Snapshot

이력서 요약은 "경험을 나열하는 곳"이 아니라 "지원자가 어떤 방식으로 일하는지 보여주는 곳"입니다.

`Candidate`에는 이름, 직무명, 연락처, 포트폴리오 링크를 정확히 넣습니다.

`Core Pitch`에는 가장 강한 한 문장을 넣습니다.

예시:

> 아이디어를 콘텐츠 포맷으로 만들고, 현장에서 완성도를 조율하며, 업로드 이후 반응까지 확인하는 PD입니다.

`Experience Focus`에는 업무 단계를 3개로 나눠 적습니다.

- 기획: 시장/채널 리서치, 콘셉트 도출, 회차 구성
- 제작: 질문지, 콘티, 촬영 동선, 출연자 커뮤니케이션
- 후반: 편집 방향, 자막 톤, 썸네일 문구, 숏폼 컷다운

### 4. 사용 가능 툴

실제로 사용할 수 있는 도구와 역량만 넣는 것이 좋습니다. 면접에서 바로 질문이 나올 수 있기 때문에, 잘 모르는 툴은 제외하는 편이 안전합니다.

현재 구성:

- Premiere Pro: 컷 편집, 자막, 오디오 정리, 숏폼 컷다운
- Photoshop: 썸네일, 간단한 이미지 보정, 배너형 그래픽
- Figma: 화면 구성, 레퍼런스 보드, 간단한 콘텐츠 구조안
- Blender: 간단한 3D 오브젝트, 텍스트/제품 연출, 기본 렌더링
- 촬영: 구도 확인, 현장 동선, 출연자 톤 조율
- 브랜드: 제품/서비스 메시지를 콘텐츠 언어로 변환
- 매니징: 크리에이터 일정, 요청사항, 커뮤니케이션 정리

툴 숙련도가 낮으면 "고급"이라고 쓰지 말고 "기본 편집 가능", "간단한 썸네일 제작 가능"처럼 정확히 표현합니다.

### 5. Target Fit

지원 회사에 맞춰 가장 자주 바꿔야 하는 섹션입니다. 회사 홈페이지, 유튜브, 인스타그램, 채용공고를 보고 회사의 사업 방향과 내 경험이 만나는 지점을 적습니다.

추천 구조:

- 회사가 하는 일: MCN, 커머스, 브랜드 콘텐츠, 커뮤니티 운영 등
- 회사가 필요로 할 PD 역량: 기획력, 출연자 커뮤니케이션, 빠른 제작, 성과 분석
- 내가 기여할 방식: 어떤 콘텐츠를 어떻게 만들 수 있는지

예시:

> Glow Up Rizz는 크리에이터의 매력을 콘텐츠, 커머스, 브랜드 협업으로 확장하는 회사입니다. 저는 크리에이터의 말투와 강점을 콘텐츠 포맷으로 정리하고, 브랜드 메시지가 자연스럽게 들어가는 숏폼 구조를 설계할 수 있습니다.

### 6. Production System

일하는 순서를 보여주는 섹션입니다. "열심히 합니다"보다 훨씬 설득력 있게 보입니다.

각 단계에는 실제 산출물을 넣으면 좋습니다.

- Insight Scan: 경쟁 채널 분석, 인기 영상 패턴, 댓글 반응 정리
- Format Design: 코너명, 회차 구성, 오프닝/클로징 구조
- Pre-Production: 대본, 질문지, 콘티, 촬영 준비물
- On-set Lead: 현장 분위기, 출연자 리액션, 촬영 우선순위
- Post Direction: 편집 리듬, 자막 톤, 썸네일, 업로드 후 피드백

### 7. Case Deck

가장 중요한 포트폴리오 증거 영역입니다. 실제 프로젝트가 없다면 학교 프로젝트, 개인 채널 분석, 가상 기획안도 가능합니다. 단, "내가 한 역할"과 "결과물"을 분명히 써야 합니다.

추천 작성 공식:

> 문제 상황 → 내가 맡은 역할 → 만든 산출물 → 결과 또는 배운 점

예시:

> 크리에이터 채널의 조회수 편차가 큰 문제를 발견했습니다. 인기 영상 20개를 분석해 반복되는 주제와 댓글 반응을 정리했고, 이를 바탕으로 3회차 숏폼 시리즈 기획안과 질문지를 제작했습니다.

기입하면 좋은 자료:

- 영상 링크
- 기획안 PDF
- 썸네일 이미지
- 대본 또는 질문지
- 조회수, 저장수, 댓글 반응 같은 성과

### 8. Resume Detail

학력, 경력, 교육, 자격, 수상 등을 간단히 정리하는 영역입니다. 길게 쓰기보다 채용 담당자가 확인하기 쉽게 정리합니다.

추천 형식:

> 2026.03 - 2026.06 / 숏폼 콘텐츠 제작 과정 수료 / 기획안, 촬영, 편집 실습

### 9. Contact

연락처는 실수 없이 넣는 것이 가장 중요합니다.

- 이메일은 자주 확인하는 주소 사용
- 전화번호는 하이픈 포함
- 영상 링크, PDF 링크, 인스타그램, 유튜브 등은 클릭 가능한 주소로 넣기

## 웹 기능 사용법

### 브라우저에서 확인하기

`index.html` 파일을 더블클릭하면 기본 브라우저에서 열립니다. 수정 후 브라우저에서 새로고침하면 변경 내용이 반영됩니다.

### 상단 메뉴

상단 메뉴의 `Resume`, `Fit`, `Works`, `Contact`는 페이지 안의 해당 섹션으로 이동하는 버튼입니다. `index.html`에서 `href="#resume"`처럼 연결되어 있고, 각 섹션의 `id="resume"` 값과 맞아야 합니다.

### 버튼 수정

첫 화면 버튼은 `hero-actions` 영역에 있습니다.

- `이력서 먼저 보기`: `#resume`으로 이동
- `포트폴리오 보기`: `#works`로 이동

외부 링크로 바꾸고 싶다면 `href="https://..."` 형태로 넣으면 됩니다.

### 툴 아이콘 툴팁

`사용 가능 툴` 아이콘에 마우스를 올리면 설명 툴팁이 뜹니다. 문구는 `data-tooltip="Premiere Pro"` 값을 수정하면 바뀝니다.

### 모바일 반응형

화면 폭이 줄어들면 섹션과 카드가 자동으로 세로 배치됩니다. 문장을 너무 길게 넣으면 모바일에서 줄바꿈이 많아지므로, 제목은 짧고 강하게 쓰는 것이 좋습니다.

### 링크 연결

연락처나 포트폴리오 링크를 클릭 가능하게 만들려면 아래처럼 작성합니다.

```html
<a href="https://example.com" target="_blank" rel="noreferrer">포트폴리오 보기</a>
```

이메일은 아래처럼 작성할 수 있습니다.

```html
<a href="mailto:yourname@example.com">yourname@example.com</a>
```

### 이미지 교체

첫 화면 이미지는 `assets/hero-prism-studio.png`입니다. 같은 이름으로 이미지를 교체하면 HTML을 수정하지 않아도 반영됩니다. 다른 파일명으로 바꿀 경우 `index.html`의 이미지 경로도 함께 수정해야 합니다.

### 지원 회사별 커스터마이징

지원 회사가 바뀔 때 전체를 다시 만들 필요는 없습니다. 아래 4곳만 바꿔도 맞춤형 포트폴리오처럼 보입니다.

- Hero 본문 1~2문장
- `Target` 회사명
- `TARGET FIT`의 회사 분석 문장
- `CASE DECK`에서 회사와 가장 가까운 프로젝트 1~2개
